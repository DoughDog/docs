title: "Design your program"
description: "Decide how your subscription program is structured before you build plans in Dough Dog"
---

Before you create plans in Dough Dog, work through the four setup decisions below. They define how your program behaves, how customers experience it, and how the assignment engine selects products each cycle.

## 1. Choose an experience

The experience controls whether customers receive the same item every cycle or a different selection each time.

- **Fixed**: You send the same item every cycle. Use this when the customer is subscribing to a specific product they want repeatedly.
- **Dynamic**: The product selection is different every time. Dough Dog's assignment engine picks a product for each cycle based on your merchandising rules.

## 2. Choose a box type

The box type controls how many items ship in each cycle.

- [**Single item box**](/platform/boxes/single-item): One item per shipment. For example, one pair of socks every month.
- [**Grouped box**](/platform/boxes/grouped): Multiple items per shipment. For example, three pairs of socks every three months.

## 3. Set the frequency

Frequency is the recurring interval between shipments, expressed as a number of months. For example, a frequency of 3 ships every three months.

## 4. Define themes (optional)

Themes are optional. If you do not define any, the assignment engine selects from all eligible products in the program.

A theme is a concept that groups products together for assignment. Themes apply only to **dynamic** subscriptions — they tell the assignment engine which pool of products a customer is eligible to receive.

A theme can represent any merchandising dimension you want to organize around, such as:

- A gender (for example, men's or women's)
- A color preference
- A flavor group preference

When themes are defined, customers are matched to a theme and the assignment engine selects from products tagged with that theme each cycle.
